# Build & Development

## Prerequisites

- **PowerShell 7+** — for running the build script
- **PS2EXE module** — auto-installed by the build script
- **Client source** — generated by `openapi-generator-cli` in the `clients/` folder

## Build Process

```powershell
.\Build-CLI.ps1 [-Clean] [-SkipPath]
```

### What the Build Does

1. **Generates alias index** — Scans all `Api/*.ps1` files across 37 services, extracts function names, computes `verb entity` aliases, and builds a PowerShell hashtable literal
2. **Concatenates source** — Reads all `.ps1` files from `Private/`, `Client/`, `Model/`, `Api/` subdirectories across all services and appends them to a `StringBuilder`
3. **Injects index** — Embeds the alias index as `$Script:AliasIndex = @{ ... }` and sets `$Script:BundledMode = $true`
4. **Appends CLI logic** — Appends the full `abs-cli.ps1` which detects `BundledMode` and skips file-based loading
5. **Compiles** — Uses PS2EXE to compile the bundled script to a .NET 4.x 64-bit EXE
6. **Packages** — Creates a zip in `dist/` containing just the EXE

### Build Parameters

| Parameter | Description |
|-----------|-------------|
| `-Clean` | Remove `bin/` and `dist/` before building |
| `-SkipPath` | Don't add `bin/` to the user PATH |

### Build Output

```
bin/
  absuite.exe           # Self-contained CLI (~15 MB)
dist/
  absuite-win-x64.zip   # Distributable (~1 MB compressed)
```

## Regenerating API Clients

When the API changes, regenerate the PowerShell clients:

```powershell
cd FenixAlliance.ABS.Client.SDKs
. .\Index.ps1
Update-SuiteApiPowershellClients -hostName "https://localhost:44388/swagger"
```

This uses `openapi-generator-cli` to generate PowerShell modules for each service from the Swagger/OpenAPI specs.

Then rebuild the CLI:

```powershell
cd FenixAlliance.ABP.SDK.PowerShell
.\Build-CLI.ps1 -Clean
```

## How BundledMode Works

The CLI has two modes:

- **Disk mode** (`$Script:BundledMode = $false`): Loads client `.ps1` files from the `clients/` folder at runtime. Used during development when running `abs-cli.ps1` directly.
- **Bundled mode** (`$Script:BundledMode = $true`): All client code is already defined in the compiled EXE. `Import-ServiceModule` only sets auth config — no file I/O.

The build script sets `$Script:BundledMode = $true` at the top of the bundled script, before any client code or CLI logic runs.

## Known Issues

### Duplicate Parameters in Generated Code

Some OpenAPI-generated model files have duplicate parameter names (e.g., `GitHubUrl` vs `GithubUrl`). When encountered, fix these in the generated source before building, or they'll cause parse errors in the compiled EXE.

### Large EXE Size

The EXE is ~15 MB because it contains all model/DTO classes for 37 services inline. The zip compresses to ~1 MB. Runtime memory usage is proportional — all functions are loaded at startup.
